# Parallel — C.W.A.S. Workspace Skill

## Overview

**C.W.A.S.** = Controller → Workers → Artifacts → Synthesis.

Use when a PLAN.md (or a goal) decomposes into 4–6 independent tasks that can execute in parallel across tmux panes. Each Worker runs Claude Code with a task brief as its system prompt; the Controller monitors done-signals and synthesizes outputs.

**When to use:**
- PLAN.md has 2–6 independent, non-blocking steps
- Fan-out work: code generation, multi-file scaffolding, parallel refactors
- User wants faster wall-clock time than serial execution

**When NOT to use:**
- Steps have hard dependencies (step B requires step A's output)
- Only 1 step (serial is simpler)
- Windows (tmux is *nix only)

## Workspace Contract

Generated under `.cks/parallel/YYYYMMDD-HHMMSS/`:

```
.cks/parallel/20260517-143105/
├── launch.sh                 # executable; opens tmux session
├── CLAUDE.md                 # Controller system prompt (opus)
├── tasks/
│   ├── worker-01.md          # per-worker brief (sonnet)
│   ├── worker-02.md
│   ├── ...
│   ├── worker-NN.md
│   └── worker-XX.done        # written by worker when done
└── src/
    ├── interfaces.md         # shared contracts (all workers read, Controller mediates writes)
    └── (worker outputs land here under worker-scoped subdirs)
```

**Rules:**
- Workspace directory is second-precision timestamped — never reuse a name
- No artifacts written outside `.cks/parallel/{ts}/`
- Workers do not modify each other's `src/` subdirs
- `interfaces.md` is immutable once workers start; to extend it, worker writes `tasks/worker-XX.blocked`

## launch.sh Template

```bash
#!/usr/bin/env bash
# Generated by /cks:parallel on {ISO_TIMESTAMP}
# C.W.A.S. workspace: 1 Controller (opus) + {N} Workers (sonnet)

set -euo pipefail

SESSION="cks-parallel-{TIMESTAMP}"
WORKSPACE="$(cd "$(dirname "$0")" && pwd)"

if ! command -v tmux >/dev/null 2>&1; then
  echo "tmux not found. Install with: brew install tmux  (macOS)  |  apt install tmux  (Linux)"
  exit 1
fi

# Kill stale session of same name (idempotent rerun)
tmux kill-session -t "$SESSION" 2>/dev/null || true

# Pane 0: Controller (opus)
tmux new-session -d -s "$SESSION" -c "$WORKSPACE" \
  "claude --model claude-opus-4-7 --append-system-prompt \"$(cat '$WORKSPACE/CLAUDE.md')\""

# Workers: split panes (sonnet)
tmux split-window -t "$SESSION" -c "$WORKSPACE" \
  "claude --model claude-sonnet-4-6 --append-system-prompt \"$(cat '$WORKSPACE/tasks/worker-01.md')\""
tmux select-layout -t "$SESSION" tiled

# ... repeat split-window + select-layout for each worker

tmux select-pane -t "$SESSION":0.0
tmux attach -t "$SESSION"
```

Notes:
- `--append-system-prompt` is the correct Claude Code CLI flag
- `select-layout tiled` balances panes as workers are added
- `kill-session ... || true` makes rerun idempotent

## Controller CLAUDE.md Template

```markdown
# Controller — Parallel Workspace {TIMESTAMP}

You are the **Controller** in a C.W.A.S. (Controller → Workers → Artifacts → Synthesis) pattern.

{IF non-deterministic mode}
> ⚠️ NON-DETERMINISTIC DECOMPOSITION
> Workers were generated from a bare goal string, not a PLAN.md. Decomposition
> may differ between runs. Review worker briefs before trusting the synthesis.
{END IF}

## Workspace
- Root: `{WORKSPACE_PATH}`
- Workers: {N}
- Shared artifacts: `src/`
- Worker briefs: `tasks/worker-XX.md`
- Shared contracts: `src/interfaces.md`

## Your Job

1. **Monitor** — every ~60s, count done markers:
   ```bash
   ls tasks/*.done 2>/dev/null | wc -l
   ```
2. **Read worker outputs** as each `worker-XX.done` appears. Outputs live under `src/`.
3. **Synthesize** when all {N} done markers exist:
   - Merge worker outputs per interfaces.md contract
   - Resolve naming/shape conflicts in favor of interfaces.md
   - Write final synthesis to `src/SYNTHESIS.md`
4. **Report** to the user in this pane when synthesis is complete.

## When all workers done, do this:
{EMITTED PER FEATURE — concrete merge instructions tied to actual artifact paths}

## Rules
- Do NOT edit worker briefs or interfaces.md once workers start
- Do NOT kill worker panes — let them complete or fail visibly
- If a worker stalls > 10min, flag it in synthesis but continue
```

## Worker Brief Template

```markdown
# Worker {XX} — {TASK_TITLE}

## Goal
{One-paragraph goal extracted from PLAN.md step or goal decomposition}

## Inputs
- PLAN.md step: {step number + title, if --from-plan}
- Shared contracts: `../src/interfaces.md` (READ — do not modify)
- Workspace root: `{WORKSPACE_PATH}`

## Expected Outputs
- `src/{output-path-1}`
- `src/{output-path-2}`

## Done Signal
When finished, write an empty marker file:
```bash
touch tasks/worker-{XX}.done
```
Do NOT write the marker until all outputs are in place and self-checked.

## Rules
- Stay within your goal — do not edit other workers' outputs
- Respect every shape declared in `src/interfaces.md`
- If you must extend interfaces.md, STOP and write `tasks/worker-{XX}.blocked`
  with a one-line reason; the Controller will mediate

{IF grouped (steps > 6)}
## Grouped Steps
This worker covers original PLAN.md steps {a, b, c}. Rationale:
{one-line grouping rationale}
{END IF}
```

## interfaces.md Template

```markdown
# Shared Interfaces — Parallel Workspace {TIMESTAMP}

All workers MUST respect the shapes, names, and file locations declared here.
Changes require Controller mediation (write `tasks/worker-XX.blocked` with reason).

{IF --from-plan AND PLAN.md has a Shared Contracts/Interfaces section}
## Extracted from PLAN.md
{verbatim copy of PLAN.md shared-contracts section}
{ELSE}
## Conventions
- File naming: `{kebab-case}.{ext}`
- Module paths: relative to `src/`
- No worker writes outside `src/{worker-scoped-subdir}/`
{END IF}

## Types / Shapes
{auto-extracted from PLAN.md when present, else stub — Controller populates before synthesis}

## Conflict Resolution
1. interfaces.md is the source of truth
2. If two workers produce conflicting shapes, Controller picks the one matching this file
3. If neither matches, Controller writes the correction and notes it in SYNTHESIS.md
```

## Worker-Count Rules

| Condition | Behavior |
|-----------|----------|
| Steps = 1 | Emit 💡 SUGGESTION (overkill); proceed only on user confirm |
| Steps 2–6 | 1:1 step → worker mapping |
| Steps > 6 | Workers 01–05 get one step each; worker-06 gets all remaining with "Grouped Steps" section |
| Bare goal | Agent decomposes into 4–6 workers |

Minimum: 2 workers. Maximum: 6 workers.

## Common Rationalizations

| Rationalization | Reality |
|---|---|
| "I'll just run launch.sh from inside the agent" | Tmux needs an interactive TTY. Agent stops at the Run: instruction. |
| "More workers = faster" | Tmux pane density caps at ~6 for readable screens. Group beyond that. |
| "Workers can edit each other's files for cleanup" | No. Filesystem-only coordination requires strict scope isolation. |
| "interfaces.md is optional if the goal is simple" | Even 2 workers can collide on a shared shape. Always emit it. |
| "Bare goal mode is fine for production work" | Non-deterministic. Use --from-plan when reproducibility matters. |
