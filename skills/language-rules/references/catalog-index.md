# Language Rules Catalog Index

Quick reference for available rule catalogs and their key rules.

| Language | File Generated | Key Rules |
|----------|---------------|-----------|
| TypeScript/JavaScript | `.claude/rules/typescript.md` | strict mode, no `any`, prefer `const`, typed errors, path aliases |
| Python | `.claude/rules/python.md` | type hints, f-strings, dataclasses, specific exceptions, isort order |
| Go | `.claude/rules/go.md` | always check errors, wrap with context, table-driven tests, context propagation |
| Rust | `.claude/rules/rust.md` | prefer borrowing, `thiserror`/`anyhow`, no `unwrap()`, iterators over loops |

## Framework Extensions

Each language catalog includes framework-specific rules when detected:
- TypeScript → React (functional components, hooks, memoization)
- Python → Django (fat models, queryset annotations)
- Go → Standard layout (`cmd/`, `internal/`)

## How Rules Are Generated

Rules are generated by `/cks:bootstrap` Step 6. The skill detects languages via config files (tsconfig.json, pyproject.toml, go.mod, Cargo.toml) and generates only relevant catalogs.
