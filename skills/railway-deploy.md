# Railway Deployment Reference

Use this file when Q8 ≠ "none". Select the config block matching the service type and adapt it.

---

## railway.toml Configs by Service Type

### Node.js / Next.js

```toml
[build]
builder = "NIXPACKS"
buildCommand = "npm install && npm run build"

[deploy]
startCommand = "npm start"
healthcheckPath = "/"
healthcheckTimeout = 30
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 3

[[services]]
name = "[PROJECT_NAME]"
```

### Python (FastAPI / Flask / Worker)

```toml
[build]
builder = "NIXPACKS"
buildCommand = "pip install -r requirements.txt"

[deploy]
startCommand = "uvicorn main:app --host 0.0.0.0 --port $PORT"
healthcheckPath = "/health"
healthcheckTimeout = 30
restartPolicyType = "ON_FAILURE"

[[services]]
name = "[PROJECT_NAME]"
```

### n8n Instance

```toml
[build]
builder = "NIXPACKS"

[deploy]
startCommand = "n8n start"
healthcheckPath = "/healthz"
healthcheckTimeout = 60
restartPolicyType = "ALWAYS"

[[services]]
name = "[PROJECT_NAME]-n8n"

[variables]
N8N_HOST = "0.0.0.0"
N8N_PORT = "$PORT"
N8N_PROTOCOL = "https"
```

### Static Site

```toml
[build]
builder = "NIXPACKS"
buildCommand = "npm install && npm run build"

[deploy]
startCommand = "npx serve dist -p $PORT"
healthcheckPath = "/"

[[services]]
name = "[PROJECT_NAME]"
```

## deploy.sh Template

```bash
#!/bin/bash
# deploy.sh — [PROJECT_NAME] Railway Deployment
# Generated by cicd-claude skill

set -e

PROJECT="[PROJECT_NAME]"
SERVICE="[RAILWAY_SERVICE_NAME]"

# ─── Help ───────────────────────────────────────────────────────────────────
if [[ "$1" == "--help" || "$1" == "-h" ]]; then
  echo ""
  echo "  deploy.sh — Deploy $PROJECT to Railway"
  echo ""
  echo "  Usage: ./deploy.sh [--env production|staging] [--dry-run]"
  echo ""
  echo "  Options:"
  echo "    --env         Target environment (default: production)"
  echo "    --dry-run     Validate config without deploying"
  echo "    --help        Show this message"
  echo ""
  echo "  Required environment variables:"
  # [Insert one line per var from Q9]
  echo "    [VAR_NAME]    [purpose]"
  echo ""
  exit 0
fi

# ─── Parse args ─────────────────────────────────────────────────────────────
ENV="production"
DRY_RUN=false

while [[ "$#" -gt 0 ]]; do
  case $1 in
    --env) ENV="$2"; shift ;;
    --dry-run) DRY_RUN=true ;;
  esac
  shift
done

echo "▶ Deploying $PROJECT to Railway ($ENV)..."

# ─── Validate env vars ───────────────────────────────────────────────────────
echo "→ Checking required environment variables..."
MISSING=()

# [Add one block per var from Q9:]
# [[ -z "${VAR_NAME}" ]] && MISSING+=("VAR_NAME")

if [ ${#MISSING[@]} -ne 0 ]; then
  echo "✗ Missing required environment variables:"
  for var in "${MISSING[@]}"; do
    echo "    - $var"
  done
  echo ""
  echo "  Set them in Railway dashboard or export locally before deploying."
  exit 1
fi

echo "✓ Environment variables OK"

# ─── Dry run ─────────────────────────────────────────────────────────────────
if [ "$DRY_RUN" = true ]; then
  echo "→ Dry run complete. No deployment made."
  exit 0
fi

# ─── Deploy ──────────────────────────────────────────────────────────────────
echo "→ Deploying to Railway..."
railway up --service "$SERVICE" --detach

echo ""
echo "✓ Deployment triggered."
echo "  Monitor: https://railway.app/project/[RAILWAY_PROJECT_ID]"
echo ""

# ─── Health check ────────────────────────────────────────────────────────────
echo "→ Waiting for health check..."
sleep 10
railway status --service "$SERVICE"
```

## Adaptation Rules

- Replace ALL `[PROJECT_NAME]`, `[RAILWAY_SERVICE_NAME]`, `[RAILWAY_PROJECT_ID]` with real values from intake
- Add one env var validation block per variable from Q9
- Add one `--help` description line per variable from Q9
- Set `startCommand` to match the actual entry point for the service type
- For n8n: always set `restartPolicyType = "ALWAYS"` — n8n must stay running
- Health check path must be a real route in the project, not a guess
- If Q8 service type isn't listed above, default to Node.js config and note the assumption
