Stop presenting options. Diagnose the situation and take the best action.

Decision framework:
1. **Safety first** — Will this break anything? If risky, warn but still recommend.
2. **Simplest solution** — Pick the approach with the least complexity.
3. **Convention over configuration** — Follow existing project patterns.
4. **Reversible over irreversible** — Prefer changes that can be undone.

Context: $ARGUMENTS

Analyze the situation, make a decision, explain it in one sentence, and execute it.
