# Compression Boundary Rules

## Two Systems, Two Surfaces

CKS ships two complementary compression systems. They operate on different surfaces and MUST NOT be applied redundantly:

| System | Surface | Trigger | Mechanism |
|--------|---------|---------|-----------|
| **Caveman** | Agent *output* — prose responses, summaries, status reports | `/cks:caveman`, `output-voice.md` | Rewrites prose: drops articles, filler, hedging |
| **Headroom** | Agent *input* — tool stdout, file reads, RAG chunks, Bash output | `/cks:headroom`, `headroom_compress` MCP tool | Compresses payload tokens before they enter the context window |

## Rules

**1. Never stack both systems on the same content.**
Compressing a chunk with `headroom_compress` and then running caveman on it produces no additional gain and risks double-mangling structured content.

**2. Caveman owns prose output. Headroom owns payload input.**
If content is going INTO the context window from an external source (Bash stdout, Read tool result, grep output, API response, RAG chunk) → headroom is the right tool.
If content is being generated BY an agent as a reply → caveman is the right tool.

**3. Headroom is optional; caveman is default ON.**
Caveman fires on every response when enabled. Headroom requires an explicit invocation (`headroom_compress`) or an agent that intentionally compresses its inputs. Agents MUST NOT auto-apply `headroom_compress` without user instruction.

**4. `headroom_retrieve` is the recovery path.**
If headroom compression loses context that turns out to be needed, call `headroom_retrieve` to restore the original. This is the only recovery mechanism — do not attempt manual reconstruction.

**5. Secrets in compressed payloads.**
Headroom may compress tool output containing credentials. Per `.claude/rules/secrets.md`, any credential that surfaces in `headroom_retrieve` or `headroom_stats` output MUST be masked before display. The binary's compression does not sanitize secrets.

## Common Rationalizations

| Rationalization | Reality |
|---|---|
| "Caveman already compresses everything" | Caveman only rewrites agent-generated prose. It cannot compress a 50KB Bash stdout. |
| "I'll run headroom on the agent's reply too" | Agent replies are caveman's domain. Stacking headroom on prose output adds latency with no benefit. |
| "headroom_compress is safe to apply automatically" | Rule 3: headroom requires explicit invocation. Auto-application changes agent behavior without user consent. |
| "I don't need headroom_retrieve — I have the compressed version" | Compression is lossy for some payloads. `headroom_retrieve` is the only lossless recovery path. |
